架构类

  • 全站 HTTPS

    出于安全和防拦截考虑，推进全站 HTTPS 的演进

    • js/css 静态资源走 https cdn，源站同样配置 https 证书

    • 收敛静态资源和接口请求域名，链路复用保证网站性能

    • 扫描 gitlab/svn 仓库的前端代码，http 请求推动开发者更改为 //，不写死协议类型

    • 代码提交前做 eslint 检查，不允许 http 提交，同时保留白名单机制，防止需调用第三方 http 服务的情况存在

    • 各系统灰度部署，根据域名、流量比例等多维护控制流量切换，保证切换过程对用户无感知，有问题可快速回滚
